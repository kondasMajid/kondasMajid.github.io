1. Could not find plugin “proposal-numeric-separator”

It is a issue with the babel dependencies conflicts between internal packages

Angular app dependency defines a broad version range of packages and their version update so you want to pin it to an earlier version

Possible solutions
npm i @babel/compat-data@7.8.0
Npm install or Yarn install
build - ng build
2. An unhandled exception occurred: Job name “..getProjectMetadata” does not exist

This happens when there is a problem with @angular-devkit/build-angular.

To fix is eithe Updating the version or downgrading to that specific version.

For me had this error after npm audit found vulnerabilities which updated it to 0.9+
npm i @angular-devkit/build-angular@0.803.25/24
ng update @angular/cli @angular/core
